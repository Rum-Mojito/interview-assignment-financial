# Example Data Scenarios

These examples are **business narratives** for interviews. In this repository, **Requirement 1** turns natural-language text into schema via **`concepts.json` / `field_packs.json` / `system_profiles.json` / `business_lines.json`** and related config (`docs/non_tech_how_to_generate_scheam.md`); the entity names below are illustrative, not a hard-coded table list.

## Scenario 1 - CRM System
A financial institution maintains a CRM system containing customer profiles, accounts, transactions, and interaction history.
Data entities may include:
Customers
Accounts
Transactions
CustomerInteractions

## Scenario 2 - Trading Platform
System contains trade orders, portfolios, market instruments, and trade executions.

## Scenario 3 - Credit Risk Dataset
Includes borrower information, loan contracts, repayment history, and risk indicators.
